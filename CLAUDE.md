# 黑马记账 — 产品文档

## 项目概述

**项目名称**：黑马记账（Heima Vibe Accounting Book）

**项目目标**：开发一款可运行在 Windows 和 Mac 电脑上的桌面记账应用，帮助用户记录每一笔人民币花销，并支持二级分类管理。

**目标用户**：有记账需求的普通用户，操作简单直观。

---

## 功能需求

### 1. 记账功能（核心）
- 用户可记录每一笔**支出**或**收入**，包含以下字段：
  - **类型**（支出 / 收入，二选一）
  - **金额**（人民币，单位：元，精确到小数点后2位）
  - **一级分类**（必选，根据所选类型动态显示对应分类）
  - **二级分类**（必选，根据一级分类动态联动）
  - **日期**（默认当天，可手动修改）
  - **备注**（选填，自由文本）
- 支持新增、编辑、删除记账记录

### 2. 记账记录查看
- 以列表形式展示所有记账记录，按日期倒序排列
- **收入用绿色显示，支出用红色显示**，便于区分
- 支持按日期范围筛选
- 支持按类型筛选（全部 / 仅收入 / 仅支出）
- 支持按分类筛选
- 支持按金额范围筛选
- 支持关键字搜索（搜索备注内容）

### 3. 统计报表
- **月度收支总览**：显示当月总收入、总支出、结余
- **支出分类占比**：各支出大类金额和占比（饼图）
- **收入分类占比**：各收入大类金额和占比（饼图）
- **每日收支明细**：每日收入和支出分别统计
- **日支出日历视图**：日历上显示每天的支出总额

### 4. 数据管理
- 数据存储在本机，无需联网
- 支持数据导出（Excel 格式）
- 支持数据备份和恢复

## 技术方案选型

> 由于你对技术不熟悉，以下列出 3 个可行方案，每个方案附有通俗解释和优劣势对比。请你仔细阅读后做出选择。

---

### 方案 A：Java 桌面应用（JavaFX + H2 数据库）

**通俗解释**：用 Java 语言写一个真正的桌面程序，就像你电脑上的记事本、计算器一样，双击图标就能打开使用。界面用 JavaFX 框架来画，数据存储在 H2（一个嵌入式的轻量数据库，自动保存在你电脑的文件里）。

| 优势 | 劣势 |
|------|------|
| 真正意义上的桌面APP，双击即可运行，体验好 | 界面美观度中等，不如网页应用精致 |
| 跨平台支持好，Windows/Mac/Linux 都能跑 | 打包后的安装包体积较大（约50-80MB） |
| 技术栈统一，只需要了解 Java 一种语言 | JavaFX 社区活跃度一般，遇到问题查找资料相对少 |
| 本地数据存储安全，不联网也能用 | Mac 上需要额外安装 Java 运行环境（JDK 17+） |
| 与当前项目的 Maven/Java 环境直接兼容 | UI 定制化需要写较多代码 |

**适合谁**：想要一个真正的桌面APP，不介意界面风格偏传统。

---

### 方案 B：Web 桌面应用（Spring Boot 后端 + Vue.js 前端 + Electron 桌面壳）

**通俗解释**：本质是一个网页应用，但用 Electron 给它套了一层"桌面壳"，让它看起来像一个桌面APP。后端用 Spring Boot 处理数据和业务逻辑，前端用 Vue.js 画界面（类似你平时访问的网站），Electron 负责打开一个窗口来显示这个网页。

| 优势 | 劣势 |
|------|------|
| 界面可以做得很漂亮，和现代网页一样精致 | 技术栈复杂，需要同时掌握 Java + JavaScript + Node.js |
| Vue.js 生态丰富，有很多现成的UI组件库可用 | 打包体积很大（约150-200MB），因为内置了Chromium浏览器 |
| 前后端分离，便于后期扩展（比如后续做手机APP可以复用后端） | 内存占用较高（运行时约200-400MB） |
| 社区非常活跃，遇到问题容易找到答案 | 项目结构复杂，对新手不够友好 |
| 图表、动画等高级UI效果容易实现 | 开发时需要启动前后端两个服务，调试麻烦 |

**适合谁**：追求界面精美，对功能丰富度要求高，并且愿意接受稍高的学习成本。

---

### 方案 C：纯 Web 应用（Spring Boot + Thymeleaf 模板 + 浏览器访问）

**通俗解释**：写一个 Web 网站，启动后在你电脑的浏览器里打开（比如 Chrome 浏览器访问 `http://localhost:8080`）。虽然它不是传统桌面APP的样子，但你可以把它收藏到浏览器书签，用起来也很方便。后端和页面都在一个项目里，结构简单。

| 优势 | 劣势 |
|------|------|
| 项目结构最简单，只需 Java 一种语言 | 不是传统桌面APP，需要在浏览器里打开 |
| 开发速度快，代码量最少 | 界面交互不如 Vue/React 流畅 |
| 不需要安装任何桌面壳（Electron等） | 浏览器标签页多了容易找不到 |
| 打包简单，一个 jar 文件就能运行 | 无法使用操作系统的原生功能（如系统托盘、全局快捷键等） |
| 学习曲线最低，最适合新手入门 | 页面切换需要刷新，体验像传统网站 |

**适合谁**：追求简单实用，不介意在浏览器里操作，希望快速看到可用产品。

## ⚠️ 关于 MySQL 的提醒

MySQL 8.0 是一个**需要单独安装的服务型数据库**（类似你电脑上要先装一个 MySQL 服务程序，APP 才能工作）。对于桌面记账应用来说，这意味着：

- **优点**：数据库功能强大，稳定可靠，符合企业级标准
- **缺点**：每台电脑都需要单独安装 MySQL 8.0，不能做到"下载即用"

如果未来你希望 APP 可以"下载即用"无需安装数据库，我们可以后续将数据库切换为 **H2** 或 **SQLite**（嵌入式数据库，数据直接存文件，无需安装）。Spring Data JPA 切换数据库只需改几行配置，毫无压力。是否接受先用 MySQL 开发？
